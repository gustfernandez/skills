# Inlined pr-finalize workflow

> **Source**: verbatim copy of `~/.claude/skills/pr-finalize/SKILL.md` (minus frontmatter).
> If you notice a discrepancy between this copy and the source, the source is authoritative — read it directly.

---

## Overview

Run this after fix commits have been pushed. Handles GitHub-side cleanup:

- Posts `"Fixed in <sha>: <summary>"` as an inline reply on each fix thread.
- Resolves each fix thread.
- Handles the gated Copilot re-request (auto for rounds 1–2; asks for round 3+).

Does NOT touch code, commit, or push.

## Inputs

- **Plan slug**: the filename stem of the plan + sidecar pair, e.g. `pr-42-bot-comments-2026-04-30`. Reads `~/.claude/plans/<slug>.actions.json`. In orchestrator autodetect mode, the slug comes from the most recent sidecar file detected in state-detection row 3.

## Step 1 — Read sidecar

```bash
cat ~/.claude/plans/<slug>.actions.json
```

Parse: `pr.owner`, `pr.repo`, `pr.number`, `worktree`, `branch`, `round_number`, `fix_threads`.

If file does not exist or is invalid JSON:
```
Error: ~/.claude/plans/<slug>.actions.json not found or invalid.
Was the plan generated by pr-comments? Check the slug.
```
Halt.

## Step 2 — Validate push

```bash
LOCAL=$(git -C <worktree> rev-parse HEAD)
UPSTREAM=$(git -C <worktree> rev-parse --abbrev-ref --symbolic-full-name @{upstream} 2>/dev/null)
REMOTE=$(git -C <worktree> rev-parse "${UPSTREAM}" 2>/dev/null)
```

If `LOCAL != REMOTE` or upstream doesn't exist:
```
Error: branch <branch> not pushed yet (or local HEAD is ahead of remote).
Push first, then re-invoke /orchestrator.
```
Halt.

```bash
SHORT_SHA=$(git -C <worktree> rev-parse --short HEAD)
```

## Step 3 — Resolve fix threads

For each entry in `fix_threads`:

1. Post reply:
   ```bash
   gh api repos/<owner>/<repo>/pulls/<N>/comments/<comment_id>/replies \
     -X POST -f body="Fixed in ${SHORT_SHA}: <summary>"
   ```

2. Resolve thread:
   ```bash
   gh api graphql -f query='
     mutation($tid: ID!) {
       resolveReviewThread(input: {threadId: $tid}) {
         thread { id isResolved }
       }
     }' -F tid=<thread_id>
   ```

On HTTP 4xx/5xx for either call: log `FAILED: <thread_id> — <http status>`, increment failure counter, and continue. Do NOT abort on individual failures. Surface all failures in the final summary.

## Step 4 — Gated Copilot re-request

Use `round_number` from the sidecar (count of Copilot reviews **before** this push round).

**`round_number <= 1`** (auto re-request):
```bash
gh pr edit <N> -R <owner>/<repo> --add-reviewer "@copilot"
```
Print: `Re-requested Copilot — round <round_number+1>, auto.`

**`round_number >= 2`** (halt — do NOT re-request):
```
Round <round_number+1> — recommend stopping. Copilot has already reviewed <round_number> times.
If you want one more round, run:
  gh pr edit <N> -R <owner>/<repo> --add-reviewer "@copilot"
```
Do not run the command. Orchestrator ends here.

## Step 5 — Summary

```
<repo>#<N>: <k>/<total> fix-threads resolved, <f> failed, commit <SHORT_SHA>, round <round_number+1>, re-requested <yes|asked>.
```

If `f > 0`, list the failed thread IDs after the summary line.

## Quick reference

| `round_number` | Re-request behavior |
|---|---|
| 0 | Auto — round 1, first Copilot review |
| 1 | Auto — round 2, reasonable iteration |
| 2 | Ask — round 3, time to consider stopping |
| ≥3 | Ask — round 4+, strong recommendation to stop |

## Common mistakes

- **Running before push.** The SHA-check in Step 2 will catch this.
- **Using databaseId for resolveReviewThread.** Must use the GraphQL node ID (`PRRT_...`).
- **Re-requesting Copilot on round 3+.** Print the command; do not call it.
- **Silently dropping failed threads.** Log each failure and surface in the final summary.
